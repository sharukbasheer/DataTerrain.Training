# Mediator Design Pattern – Complete & Detailed Explanation (C#)

---

## 📌 Overview

The **Mediator Design Pattern** is a **Behavioral Design Pattern** that centralizes communication between objects.

---

## ❓ Why the Mediator Pattern Exists

### 🔴 The Problem

Direct object communication leads to:
- Tight coupling
- Complex dependencies

---

## ✅ What the Mediator Pattern Solves

Mediator:
- Centralizes communication
- Reduces object dependencies

---

## 🎯 Goal of the Mediator Pattern

To reduce coupling between interacting objects.

---

## 🧠 Core Idea

> **“Objects communicate through a mediator.”**

---

## 🧩 Step-by-Step Explanation

1. Define mediator interface
2. Implement concrete mediator
3. Register colleague objects
4. Delegate communication

---

## 🔁 Runtime Behavior Summary

- Colleagues send messages
- Mediator routes communication

---

## ⚖️ Pros and Cons

### ✅ Advantages
- Reduced coupling
- Centralized control

### ❌ Disadvantages
- Mediator may grow complex

---

## 🎯 When to Use

- Complex object interactions
- UI component communication

---

## 🚫 When NOT to Use

- Simple interactions

---

## 🏢 Real-World Use Cases

- Chat rooms
- UI controllers
- Workflow coordinators
